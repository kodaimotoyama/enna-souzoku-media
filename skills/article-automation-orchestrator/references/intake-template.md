# Intake Template

Use this on first invocation when the user has not yet provided a concrete article brief.

Ask in Japanese by default:

1. 何についての記事を作りますか。
2. 誰向けの記事ですか。
3. 目的は何ですか。
4. 1記事あたりの文字数はどのくらいにしますか。
5. 何本作りますか。
6. 欲しい成果物は何ですか。タイトル案、見出し、本文、HTML、CSV など。
7. 文体やトーンに指定はありますか。
8. 必ず入れたい要素と避けたい表現はありますか。
9. CTA や誘導先はありますか。
10. 参照すべき既存記事やキーワード一覧はありますか。

If the user is in a hurry, compress this into a short form:

- テーマ
- 読者
- 目的
- 文字数
- 本数
- 成果物
- トーン
- CTA

After the answers, summarize them like this:

```text
確認した内容:
- テーマ:
- 読者:
- 目的:
- 文字数:
- 本数:
- 成果物:
- トーン:
- CTA:
- 制約:
```
